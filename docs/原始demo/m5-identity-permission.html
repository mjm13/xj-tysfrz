对账治理定位：本平台不参与权限申请审批，权限授予仍由源头系统（一卡通、门禁、网络中心、访客系统等）执行。本页面定义授权基线规则—— 默认授予 / 可申请授权 / 不予授权，平台定期拉取源头实际数据进行比对，发现越权 / 僵尸账号 / 规则缺失等异常并推送处置。

对账数据源对接：本平台通过定时拉取以下源头系统的实际授权数据进行对账。任何源头变更都在 T+1 内反映到对账结果。
公共权限主数据：本平台管理的全部校园公共权限项。新权限可通过新增录入，已有权限项不允许删除，仅可启用/停用来控制是否参与对账。停用的权限项历史数据保留可查。

检查任务配置：每个任务定义一次"源头数据 ⇔ 权限分类表"的对账行为：选定数据源、选用已启用的规则集、配置执行时间（cron）和通知策略。任务执行后产出"对账异常"和"僵尸账号"清单。
检查规则库：规则定义"什么算异常"。新规则可通过新增录入，已有规则仅支持启用/停用。启用后该规则对所有人员分类生效，不可按分类区分。这是为了保证治理一致性——避免出现"A 类查越权但 B 类不查"的合规盲区。
执行情况：所有检查任务的运行日志。可查看每次执行的耗时、处理记录数、发现的异常数、错误信息。失败任务支持重试或查看错误堆栈。
