# 06 — Demo 逻辑疑问与断点台账

> 来源：逐页精读 `docs/原始demo/` 全部 12 个 HTML + 2 个 JS（2026-06-19）。  
> 用途：每条疑问在 [05-demo-feature-tasks.md](./05-demo-feature-tasks.md) 中被功能任务引用；**实现对应页面前必须先确认 🔴 项**。

## 图例

| 严重度 | 含义 |
| --- | --- |
| 🔴 阻断 | 逻辑断裂 / 无法闭环 / 影响领域建模，开发前必须产品确认 |
| 🟡 待澄清 | 数据口径或职责边界不清，需在 design 阶段定义 |
| 🟢 瑕疵 | Demo 实现 bug 或文案问题，Vue 化时顺手修正即可 |

| 类型 | 含义 |
| --- | --- |
| 闭环 | 流程有入口无出口 / 有动作无结果 |
| 断点 | 链接/按钮无目标或无实现 |
| 数据 | 跨页数据口径/数量不一致 |
| 边界 | 职责/语义/术语不清 |
| 缺陷 | HTML 结构或脚本 bug |

## QG — 全局 / 跨页

| 编号 | 严重 | 类型 | 问题 | 建议 |
| --- | --- | --- | --- | --- |
| QG-01 | 🔴 | 边界 | **两套「源头」术语混淆**：人员采集源（source-maintenance / etl-monitor）vs 权限数据源（m5「数据源对接」），无统一术语、无互链 | 领域建模时拆分为「采集源 DataSource」与「权限系统 PermissionSystem」两个概念 |
| QG-02 | 🟡 | 边界 | **双主页**：`页面主页.html` 与 `platform-v2-home.html` 内容完全重复，v2 无任何入口引用 | 确认保留哪个为正式首页，另一个归档 |
| QG-03 | 🟢 | 断点 | 顶栏「主页」在主页自身为 `#`，子页为 `页面主页.html`，行为不一致 | Vue 路由统一为 `/` |
| QG-04 | 🔴 | 数据 | **UID 体系不统一**：m1/m3/m5/m6 用 `NP00000001`，m7 用 `UID-A00001`，跨模块无法按 UID 关联 | 全平台统一唯一人员 ID 规范 |
| QG-05 | 🟡 | 数据 | **人员总数口径冲突**：主页 94,283（不含校友）；m1 `SOURCE_CATEGORIES` 含校友合计 132,483；页头「5 权威来源」vs 筛选「7 数据源」 | 明确「在档身份」是否含校友/校外，统一来源数 |
| QG-06 | 🟡 | 断点 | **侧栏深链失效**：m6 / etl / source-maintenance 子项多指向同一 URL，无法直达指定子视图（无 hash/参数） | Vue 用子路由，每个视图独立 path |
| QG-07 | 🟢 | 闭环 | 「系统服务」能力（数据字典 / API 服务 / 系统配置）全站标「规划中」，无落地页 | 列为 Phase 后续，占位路由 |
| QG-08 | 🔴 | 闭环 | **三套源头/同步并存且互不连通**：m1 内嵌 `#view-sync`/`#view-source` + `etl-monitor.html` + `source-maintenance.html`，职责重叠；source-maintenance 注册的数据源不被 ETL 消费 | 设计统一「数据接入」上下文，明确配置→采集→入库链路 |
| QG-09 | 🟢 | 边界 | 命名不统一：自定义标签（「标签身份」/「标签管理」/「自定义分类标签」）、外协人员/外协类人员 | 统一术语表（ubiquitous-language） |
